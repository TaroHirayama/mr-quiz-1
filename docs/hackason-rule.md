# 4th Agentic AI Hackathon with Google Cloud - Rules

## Overview

**Theme**: "Evolving with AI. Create your own 'Agent'."

A hackathon focused on the latest trends in autonomous AI. Features the latest "Gemini 3" model and AI coding tools.

## Schedule

| Phase | Period |
|-------|--------|
| Registration & Development & Submission | December 10, 2025 - February 15, 2026 |
| 1st Round Review | February 16 - 23, 2026 |
| 2nd Round Review | February 24 - March 2, 2026 |
| Final Review & Announcement | March 19, 2026 |

## Prize Money

**Total: 1.75 Million JPY**

| Award | Amount | Count |
|-------|--------|-------|
| Grand Prize | 500,000 JPY | 1 |
| Excellence Award | 250,000 JPY | 3 |
| Encouragement Award | 100,000 JPY | 5 |

## Eligibility

- Individuals or teams aged 18+ residing in Japan
- Individual or team participation allowed
- For team participation, all members (including the representative) must register
- Participation in multiple projects is allowed

### Not Eligible

- Government officials
- Sponsor employees
- Individuals participating as company representatives

## Required Development Conditions

### Google Cloud Compute Products (Select at least one)

Must use one of the following:

- App Engine
- Compute Engine
- Google Kubernetes Engine (GKE)
- Cloud Run
- Cloud Functions
- Cloud TPU / GPU

### AI Technology (Select at least one)

Must use one of the following:

- Vertex AI
- Gemini API
- Gemma
- Imagen
- Agent Builder
- Agent Development Kit (ADK)
- Other AI-related APIs

## Submission Requirements

### 1. Public GitHub Repository URL

- Repository must remain **public until March 2, 2026**

### 2. Deployed URL

- **Deployment to Google Cloud is mandatory**

### 3. Zenn Article

Article settings:
- Category: `Idea`
- Topic: `gch4`

Article must include:
- Project overview
- System architecture diagram
- Demo video (~3 minutes, embedded from YouTube)

## Judging Criteria

### 1. Novelty of the Problem

- Has an unsolved practical problem been identified?

### 2. Effectiveness of the Solution

- Does the solution effectively solve the problem?

### 3. Implementation Quality and Scalability

- Implementation completeness
- Tool utilization
- Operability
- Cost-effectiveness
